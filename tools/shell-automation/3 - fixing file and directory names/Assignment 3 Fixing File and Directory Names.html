Assignment 3: Fixing File and Directory Names

Assignment Objective

Students must modify and complete the provided student-example-p3 script to rename files and directories that contain spaces in their names by replacing the spaces with dashes.

For example: A file named "A B" would be renamed to "A-B"

Script Functionality

The completed script will:

  • Work on a directory specified on the command line
  • Recursively process all files and directories within that directory
  • Replace spaces with dashes in file and directory names
  • Handle multiple consecutive spaces by converting them to multiple dashes

Command Line Options

The script supports the following command line options:

Command Line Options for fix-spaces script
Option Description
-f Only rename files (not directories)
-d Only rename directories (not files)
directory-name The name of the directory to process (last argument)

Note: Both -d and -f may be specified together. At least one of these options must be provided.

Example Usage and Results

Before Processing

Consider the following directory structure:

./C D 
./C D/E F 
./C D/E F/G H 
./C D/E F/G H/J    K 
./C D/E F/G H/J    K/a b 
./C D/E F/G H/a b 
./C D/E F/a b 
./C D/a b

Command Execution

fix-spaces -f -d "C D"

After Processing

The directory structure becomes:

C-D 
C-D/E-F 
C-D/E-F/G-H 
C-D/E-F/G-H/a-b 
C-D/E-F/G-H/J----K 
C-D/E-F/G-H/J----K/a-b 
C-D/E-F/a-b 
C-D/a-b

Important Note About Space Replacement

Each space character should be converted to exactly one dash character:

  • "a b" (one space) becomes "a-b" (one dash)
  • "a b" (three spaces) becomes "a---b" (three dashes)
  • "J K" (four spaces) becomes "J----K" (four dashes)

Functions You Must Implement

1. my_rename() Function

This function must rename a single file or directory from $1 to $2.

Required Error Checking:

  1. Check if the directory where $1 resides is writable
  2. Check if $2 already exists (don't overwrite existing files)
  3. Check the exit status of the mv command and report any errors

Critical Implementation Details:

  • Work only on the basename (last part of the path)
  • Use the provided basename() and pathname() functions
  • Only attempt to rename if the basename actually contains spaces
  • Handle path construction correctly when pathname() returns "."

Why You Must Separate Pathname and Basename

Consider a file path like: "./My Documents/Project Files/data file.txt"

You should only rename based on the basename "data file.txt", not the entire path. The directory structure "./My Documents/Project Files/" should remain unchanged for this particular file operation.

Wrong Approach - DO NOT DO THIS:
for dir in $(find_dirs "$1")

This approach will fail because:

  • Paths with spaces will be split incorrectly by the shell
  • A path like "./My Dir/Sub Dir" becomes three separate arguments: "./My", "Dir/Sub", and "Dir"
  • Your script will try to operate on non-existent partial paths
Better Approach:

Use a while loop with proper input redirection to handle paths containing spaces correctly. This ensures each complete path is processed as a single unit.

Why Quoting is Critical:

Always quote variables that contain file paths. Without quotes:

  • mv $oldpath $newpath - WRONG: Spaces cause word splitting
  • mv "$oldpath" "$newpath" - CORRECT: Preserves spaces in paths

2. fix_dirs() Function

This function must:

  • Call find_dirs() to get a list of directories with spaces in their names
  • For each directory returned, create the new name with spaces replaced by dashes
  • Call my_rename() with the old and new paths

3. fix_files() Function

This function must:

  • Call find_files() to get a list of files with spaces in their names
  • For each file returned, create the new name with spaces replaced by dashes
  • Call my_rename() with the old and new paths

Important: Understanding Find Function Output

The find_dirs() and find_files() functions return full paths, not just basenames. You must extract the basename and directory parts correctly using the provided utility functions.

Additional Error Handling Requirements

Your script must handle these error conditions by printing an appropriate error message and exiting:

  • No directory specified on command line
  • The specified directory is the current directory (.)
  • The specified directory is the parent directory (..)
  • Neither -f nor -d options are specified
  • The specified directory does not exist

Why Depth-First Processing Matters

The script uses the -depth option in the find commands. This ensures that:

  • Contents of directories are processed before the directories themselves
  • Parent directories are renamed after their contents
  • This prevents invalid path issues that would occur if directories were renamed before their contents

Key Points for Success

  • Always quote variables that may contain spaces
  • Test your basename logic separately before integrating
  • Remember that find functions return full paths, not basenames
  • Only rename files/directories whose basenames actually contain spaces
  • Handle the case where pathname() returns "." correctly

    touch "$base_dir/E F/another_normal"

    find "$base_dir" -print | sort
}
