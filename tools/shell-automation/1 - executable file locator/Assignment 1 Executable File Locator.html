Assignment 1: Executable File Locator

Overview

Write a shell script to locate executable files. This script takes a list of file names from the command line and determines which would be executed had these names been given as commands.

Goal: Create a custom implementation of the Unix which command that understands PATH variable processing and command resolution.

Learning Outcomes

  • Process colon-separated environment variables
  • Handle command-line parameters and flags
  • Implement conditional logic using test statements
  • Work with the test command to determine file properties
  • Understand shell command resolution mechanics

Requirements

Search Path Requirements

Prohibited Commands: You shall NOT use any of the following:
  • Unix which command
  • ksh whence (type) command
  • locate command
  • bash type command
  • UNIX ls command to determine if file is executable or exists
  • Any temporary files

PATH Variable Handling

The current directory can be specified in PATH by:

  • Leading colon: :/usr/bin:/bin
  • Double colon: /usr/bin::/bin
  • Trailing colon: /usr/bin:/bin:
  • Explicit dot: /usr/bin:.:/bin

Script Behavior

  • Find only first occurrence of the file unless -a flag is used
  • Display "command NOT FOUND" for unfound files
  • If command contains /, check directly without using PATH
  • No temporary files allowed
  • Script must be executable

Important Notes

  • Directories are usually executable but should not be considered valid commands
  • Use bash string comparison ([[ $path == */* ]]) or case statement to check if a file is a pathname
  • Use test command (-x file) to check if file is executable
  • Use test command (-d file) to check if path is a directory
  • Files must be both readable and executable to be valid commands

Usage Examples

Basic usage:
prompt> mywhich ls
/bin/ls
Find all occurrences:
prompt> mywhich -a cc
/bin/cc
/usr/ucb/cc
Command not found:
prompt> mywhich fooblar
