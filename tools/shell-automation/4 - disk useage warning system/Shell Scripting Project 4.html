Shell Scripting Project 4

Create a shell script on a Linux System preferable, cyberserver.uml.edu called fs_usage_warning that sends email to users when a file system is above a certain disk usage percentage. Check against the Use% of the df command.

Syntax

fs_usage_warning [-c capacity-percentage] email-address [email-address] ...
  • You can specify more than one email address separated by spaces.
  • The email address can be an account name
  • If no email addresses are specified, the program should exit with an error.
  • If the -c capacity-percentage is not specified, it should default to 60%. Do not include the % symbol when specifying the capacity percentage.

Email Messages

  • If the file system usage is less than the specified amount, no email is sent.
  • If the file system usage is greater than or equal to the specified amount and less than 90%, the email subject should be: 
    "Warning: the file system <file system directory> is above <capacity> % used"
    The email body should include the complete disk usage line from the df command for that file system along with the header. No other file systems should be in the email.
  • If the file system usage is greater than or equal to 90%, the email subject should be: 
    "Critical Warning: the file system <file system directory> is greater than or equal to 90% capacity"
    The email body should include the complete disk usage line from the df command for that file system along with the header. No other file systems should be in the email.

Requirements

  • Use the mail (mailx) command and in-line redirection (here-document) where the body of the mail message is in the here-document. Temporary files are not allowed. Using a temporary file or not using in-line redirection will result in a loss of points.
  • A separate email message should be sent for each file system that meets the warning capacity requirements.
  • This program is to be written, executed, and tested on a linux system preferably, cyberserver.uml.edu.
  • Use the df command and process the output. Only process these file system types: ext4 and ext2.

df Command Example

df -k -t ext4 -t ext2  
Filesystem                    1K-blocks     Used Available Use% Mounted on
/dev/mapper/cyberserver-root 141786532  5874056 128816568   5% /       
/dev/sda1                       233191    49878    170872  23% /boot

Helpful Tips

When processing the df command output, consider using:

  • read and while loops to iterate through the lines
  • cut command to extract specific fields, e.g.: 
    cut -d' ' -f5,6
    to get the Use% and Mounted on columns
  • awk command for more advanced parsing, e.g.: 
    df -k -t ext4 -t ext2 | awk 'NR>1 {print $5, $6}' | while read usage mountpoint; do
    # Remove trailing % from usage
    usage=${usage%\%}
    
    # Check usage against threshold  
    if ((usage >= threshold)); then
        # Send email
    fi  
done
