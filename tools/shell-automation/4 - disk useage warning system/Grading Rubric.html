Shell Programming Assignment 4 - Grading Rubric

File System Usage Warning Script (fs_usage_warning)

Point Distribution Overview

  • Functionality & Logic: 40 points
  • Command Line Argument Handling: 20 points
  • Email Implementation: 20 points
  • Code Quality & Best Practices: 15 points
  • File Requirements: 5 points

1. Functionality & Logic (40 points)

Criteria Points Excellent Good Needs Improvement
df Command Usage 10 Correctly uses df -k -t ext4 -t ext2 and processes only ext4/ext2 filesystems Uses df command correctly but minor issues with filesystem filtering Incorrect df usage or processes wrong filesystem types
Usage Percentage Extraction 10 Accurately extracts Use% field and removes % symbol correctly Extracts usage with minor parsing issues Fails to extract usage percentage or incorrect field parsing
Threshold Comparison Logic 10 Correctly compares usage against threshold and 90% critical level Comparison logic mostly correct with minor edge cases Incorrect comparison logic or missing critical threshold
Multiple Filesystem Handling 10 Processes all filesystems and sends separate emails for each that meets criteria Handles multiple filesystems with minor issues Only processes one filesystem or combines multiple in single email

Criteria Points Excellent Good Needs Improvement

File Type & Submission 2 Submitted as shell script file, not document format Correct file type Submitted as Word/PDF or other incorrect format

Common Deductions

Critical Issues (Major Point Loss)

  • -15 points: Uses temporary files instead of here-document
  • -10 points: Script doesn't execute or has major syntax errors
  • -10 points: Processes filesystem types other than ext4/ext2
  • -8 points: Sends single email for multiple filesystems

Minor Issues (Partial Point Loss)

  • -3 points: Hardcoded email addresses in script
  • -2 points: Inconsistent or missing shebang line
  • -2 points: Poor variable naming or lack of quoting
  • -1 point: Minor formatting issues in email content

Testing Criteria

Script should be tested with:

  • Default threshold (no -c option)
  • Custom threshold with -c option
  • Multiple email addresses
  • Filesystems below threshold (no email sent)
  • Filesystems above threshold but below 90%
  • Filesystems at or above 90%
  • Error conditions (no email addresses, invalid options)
Total Points: 100
