Critical Violations (Automatic Major Deductions)

  • Use of temporary files: -10 points - Assignment explicitly prohibits temporary file usage
  • Script not executable: -5 points - Basic requirement for shell script functionality
  • Major logic errors: -5 to -8 points - Fundamental algorithmic or implementation problems

Other Major Deductions:

  • -10 points: Uses for loop instead of while loop for find output
  • -10 points: Doesn't properly separate pathname and basename
  • -8 points: Missing required error condition checks
  • -5 points: Incorrect shebang line for system

Minor Deductions:

  • -3 points: Inconsistent variable quoting
  • -3 points: Poor error message formatting
  • -2 points: Missing comments or poor code organization
  • -1 point: Minor spacing or style issues

Specific Test Cases for Grading

Functionality Tests (Must Pass):

  1. Basic file renaming: Single files with spaces
  2. Basic directory renaming: Single directories with spaces
  3. Nested structure: Complex directory tree with mixed spaces
  4. Multiple spaces: Files/dirs with consecutive spaces (→ multiple dashes)
  5. Mixed content: Some items with spaces, some without
  6. Option combinations: -f only, -d only, -f -d together

Error Condition Tests (Must Handle):

  1. No arguments: Should show usage and exit
  2. No directory specified: Script with -f but no directory
  3. Current directory (.): Should reject and exit
  4. Parent directory (..): Should reject and exit
  5. No flags: Directory specified but no -f or -d
  6. Non-existent directory: Should report error and exit
  7. Permission issues: Read-only directory handling
  8. File already exists: Target filename already exists

Common Student Mistakes (Watch For)

  • For loop usage: Using for item in $(find_dirs) instead of while loop
  • Quoting errors: mv $old $new instead of mv "$old" "$new"
  • Full path renaming: Trying to rename entire path instead of just basename
  • Unnecessary renames: Attempting mv even when basename has no spaces
  • Path construction: Incorrect handling when pathname() returns "."
  • Error checking: Missing permission checks or exit status validation
  • Space replacement: Using tr or sed instead of parameter expansion
