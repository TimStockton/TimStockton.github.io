Assignment 1: Executable File Locator

Grading Rubric (100 points)

Important: This assignment tests your understanding of how shell command resolution works, particularly PATH variable processing. The grading emphasizes correct syntax/basic functionality and PATH handling as the core competencies.

Core Functionality

Total: 25 points

Basic Program Execution: 25 points

  • Syntax correctness and basic functionality: Your script must run without syntax errors and produce correct output for basic commands
  • Tested with: Standard commands like vi, tar, ls in various PATH configurations
  • Critical requirement: If your script has syntax errors or fails basic command location, you will lose most of these points

Advanced Features

Total: 75 points

-a Flag Implementation: 10 points

  • Requirement: The -a flag must find ALL occurrences of a command in PATH, not just the first one
  • Testing: Script will be tested with -a tar and similar commands
  • Expected behavior: Should return multiple paths if the command exists in multiple PATH directories

PATH Variable Handling: 40 points

Your script must correctly interpret all variations of how the current directory can be specified in PATH:

PATH Format Variations and Point Values
PATH Format Example Points Description
Colon at start :/usr/bin:/bin 10 points Leading colon means current directory is first in search path
Double colon /usr/bin::/bin 10 points Empty component between colons represents current directory
Colon at end /usr/bin:/bin: 10 points Trailing colon means current directory is last in search path
Explicit dot /usr/bin:.:/bin 10 points Explicit dot (.) represents current directory
Critical Test: The primary test for each PATH variation is whether your script can find itself when the current directory is included in PATH using that format.

Argument and Path Processing: 25 points

Multiple Arguments: 10 points

  • Requirement: Process each command-line argument as a separate command
  • Testing: Commands like script tar ls vi should return the path for each command on separate lines
  • Expected behavior: Each argument should be processed independently

Path Type Handling: 15 points

  • Absolute path handling: 5 points
    • Commands like /usr/bin/ls should be checked directly, not searched in PATH
  • Relative path handling: 5 points
    • Commands like ./script should be checked directly in the current directory
  • Directory detection and handling: 5 points
    • Directories should never be considered executable commands, regardless of permissions

Additional Requirements

(No additional points, but required for full credit)

File Validation

  • Permission checking: Files must be executable to be considered valid commands
  • Error handling: Proper "not found" messages for commands that don't exist
  • Directory vs file distinction: Directories should not be reported as executable commands

Special Cases

  • Directory with absolute path: /usr/bin should return "not found"
  • Directory with relative path: ./dirname should return "not found"
  • Non-executable files: Should return "not found" even if they exist

Critical Violations (Automatic Deductions)

  • Use of temporary files: -10 points (Assignment explicitly prohibits temporary file usage)
  • Script not executable: -5 points (Basic requirement for shell script functionality)
  • Major logic errors: -5 to -8 points (Fundamental algorithmic or implementation problems)

Testing Methodology

Your script will be tested with various PATH configurations and command combinations. The self-reference test (script finding itself) is the primary indicator of correct PATH handling for each variation.

Key Test: If your script cannot find itself when the current directory is in PATH (in any of the four formats above), you will lose points for that specific PATH handling requirement.

Key Requirements Summary

  1. When PATH contains :, ::, trailing :, or . - your script must recognize these as including the current directory
  2. Commands containing / should bypass PATH and be checked directly
  3. Directories should never be considered executable commands, regardless of permissions
  4. Multiple arguments should be processed as separate commands
  5. The -a flag should find ALL occurrences, not just the first
  6. Use proper "not found" error messages for commands that don't exist
Remember: The shell's PATH variable is fundamental to how command execution works in Unix/Linux systems. Understanding these concepts will be crucial for future assignments and real-world shell scripting.
