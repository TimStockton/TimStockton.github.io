this document provides the steps to install and set up owncloud on a server
owncloud is like self-hosted dropbox, but you have control over every aspect

-------------------------
install LAMP stack and owncloud
-------------------------
tutorial, how to install on ubuntu 24.04:
https://www.youtube.com/watch?v=jfWZoD4hwqI

start by updating system packages:
apt update
apt upgrade

install required dependencies (LAMP stack):

apt install apache2

apt install mariadb-server
mysql_secure_installation
enter root password
switch to unix socket auth
do not change root pass
remove anonymous users
disallow remote login for root
remove test database
reload privilege tables

install repository, php, and extensions:
apt-get install software-properties-common
add-apt-repository ppa:ondrej/php
apt update
PHP 7.4 is EOL, but ownCloud Server officially supports PHP 7.2 / 7.4
run this command to install PHP 7.4 and it's extensions from the repo:
apt install php7.4 libapache2-mod-php7.4 \
php7.4-mysql php7.4-xml php7.4-curl php7.4-gd \
php7.4-mbstring php7.4-zip php7.4-intl php7.4-bz2

apt update

set up maria db:
mysql
CREATE DATABASE owncloud;
CREATE USER 'owncloudadmin'@'localhost' IDENTIFIED BY 'strongpassword';
GRANT ALL PRIVILEGES ON owncloud.* TO 'owncloudadmin'@'localhost';
FLUSH PRIVILEGES;
SHOW DATABASES;
SELECT user FROM mysql.user;
EXIT;

apt install zip unzip bzip2 curl

download latest version of owncloud:
https://owncloud.com/download-server/
Version: ownCloud 10.16.0
place the zip file in /tmp (using winSCP or FTP as needed)
unzip latest-owncloud-version.zip
move extracted directory to apache root directory:
mv owncloud /var/www/html
set ownership and permissions:
cd /var/www/html/owncloud
find . -type d -exec chmod 755 {} \;
find . -type f -exec chmod 644 {} \;
chown -R www-data:www-data /var/www/html/owncloud
^ www-data is apache

configure apache to serve owncloud:
vim /etc/apache2/sites-available/owncloud.conf

<VirtualHost *:80>
    ServerAdmin admin@example.com
    DocumentRoot /var/www/html/owncloud
    ServerName your_domain_or_ip

    <Directory /var/www/html/owncloud/>
        Options +FollowSymlinks
        AllowOverride All

        <IfModule mod_dav.c>
            Dav off
        </IfModule>

        SetEnv HOME /var/www/html/owncloud
        SetEnv HTTP_HOME /var/www/html/owncloud

    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/owncloud_error.log
    CustomLog ${APACHE_LOG_DIR}/owncloud_access.log combined

</VirtualHost>

enable new config and required modules:
a2ensite owncloud.conf
a2enmod rewrite headers env dir mime ssl
systemctl restart apache2

go to domain/ip address for owncloud user interface:
http://domain-or-ip/owncloud/index.php
use the IP address for apache

create admin account via interface
username: owncloudadmin
password: securepassword
data folder: /var/www/html/owncloud/data
database user: owncloudadmin
database password: securepassword
database name: owncloud
database host: localhost

can later enable SSL encryption, but a domain is required for SSL

if needed, change which version of php apache is using:
a2dismod php8.2
a2enmod php7.4
sudo update-alternatives --set php /usr/bin/php7.4
check the php version:
php -v
restart apache:
systemctl restart apache2

-------------------------
create additional user
-------------------------
log in as the admin user
navigate to users
add a new user
Take a screenshot of the Users page showing both accounts.

-------------------------
configure email server
-------------------------
in admin settings, find email server configuration
enter SMTP details (gmail)
send mode: SMTP (having owncloud relay mail externally)
server address: smtp.gmail.com
port: 465 (uses SSL encryption)
encryption type: SSL/TLS
authentication: on
auth credentials: gmail-username and app-password
save the configuration
run the “Send test email” button

-------------------------
test owncloud setup
-------------------------
log in as the new user (not admin)
upload any file (PDF, text file, etc)
download the same file
